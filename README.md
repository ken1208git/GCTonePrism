# ゲームセンターTONE 統合ランチャーシステム「Prism」

## 👋 GCTonePrismへようこそ！

このプロジェクトは、文化祭でPC部が制作したゲームを、一つのアプリケーションから起動・展示するための統合ランチャーです。
このドキュメントは、プロジェクトに関わるすべての部員（ゲーム制作者、プログラマー）のための公式な説明書です。

**このガイドを読んで、もし分からない言葉や、うまくいかないことがあれば、一人で悩まず、気軽に先輩や顧問の先生に質問してください！**

*   **ゲームを展示したい方へ:** 下にある「🎮 あなたのゲームをPrismに載せよう！」のセクションを読めば、誰でも簡単に自分のゲームをランチャーに追加できます。
*   **開発に参加したい方へ:** このドキュメント全体を読むことで、プロジェクトの全体像と開発の進め方を理解できます。

## ✨ 主な機能

*   **統一されたゲーム体験:** 部員が様々なエンジン（Unity, Godot, RPGツクールなど）で作ったゲームを、一つの美しいインターフェースから起動でき、文化祭の展示に一体感を生み出します。
*   **動的なビジュアル:** PS5のホーム画面にインスパイアされた、リッチなUIを採用。ゲームを選択すると、背景がそのゲームのアートワークに変化し、来場者をゲームの世界観に引き込みます。
*   **制作者に優しい設計:** プログラミングの知識がなくても、簡単なテキストファイル（JSON）を編集するだけで、自分のゲームをランチャーに追加できます。
*   **レスポンシブ対応:** WQHDやフルHDなど、どんな解像度のモニターで表示しても、レイアウトが崩れることのない、柔軟な画面設計を採用しています。
*   **開発者向けツール:** `Ctrl + Alt + F12`で呼び出せるデバッグモニターを搭載し、開発とテストを強力にサポートします。

## 🎮 あなたのゲームをPrismに載せよう！ (文化祭への出品方法)

あなたの力作を、Prismに載せてみませんか？ 手順はとても簡単です！

### ⚠️ ステップ0：絶対に守るべき「命名規則」

プログラムを安定して動かすため、以下の**2つ**の名前には、**半角の英数字とハイフン `-` 、アンダーバー `_` のみ**を使用してください。
**ひらがな、カタカナ、漢字、スペース、その他の特殊記号は、絶対に含めないでください。**

1.  **あなたのゲームのフォルダ名**
2.  **あなたのゲームの実行ファイル名 (`.exe`)**

**ご安心ください！**
これらの名前は、あくまでプログラムがファイルを識別するためのものです。
**ランチャーの画面に表示されるゲームの正式名称は、後述する`title`項目で、日本語を使って自由に、そして華やかに設定できます。**

### ステップ1：ゲームをビルドし、展示用の場所で動作確認する

1.  **ゲームを書き出す（ビルドする）**
    *   あなたが使っているゲームエンジン（Unity, Godot, RPGツクールなど）で、ゲームを**Windows向けに書き出し**してください。
    *   **（注意）** 各エンジンでの具体的な書き出し方法は、ここでは説明しきれません。分からない場合は、**各自でWeb検索などを使って調べてみましょう。**

2.  **展示用のゲーム置き場に、あなたのゲーム専用フォルダを作る**
    *   まず、部内の共有フォルダにある、**展示専用**の以下の場所を開いてください。
        **`共有フォルダ(PCクラブ)→【ゲームセンターTONE】→Games\`**
    *   この`Games`フォルダの中に、上記の命名規則に従って、**あなたのゲーム用の新しいフォルダ**を作成します。（例: `galaxy_striker`）

3.  **書き出したゲームをコピーする**
    *   ステップ1で書き出した`.exe`ファイルや関連ファイルを**すべて、今作ったあなたのゲームフォルダの中にコピー**してください。

4.  **【最重要】展示用の場所で、最終動作確認をする**
    *   コピーが完了したら、**必ず、ステップ2で作成したあなたのゲームフォルダの中にある`.exe`ファイルを直接ダブルクリックして、**ゲームが問題なく起動し、正常に動作するかを確認してください。
    *   （この最終確認をすることで、「開発フォルダから直接実行した時しか動かない」といった、ファイルの場所に関するトラブルを完全になくすことができます。）

### ステップ2：設定ファイル `launcher_info.json` を作成する

次に、あなたのゲームフォルダの中に、`launcher_info.json`という名前で新しいテキストファイルを作成し、以下の**テンプレート**を**すべてコピー**して貼り付けてください。

```json
    {
      "game_id": "ここに半角英数字のIDを入力 (フォルダ名と同じ)",
      "title": "ここにゲームの正式名称を入力",
      "description": "ここにゲームの紹介文を入力してください。",
      "developers": [
        { "last_name": "姓", "first_name": "名", "grade": 47 }
      ],
      "release_year": 2025,
      "genre": ["ジャンル1", "ジャンル2"],
      "min_players": 1,
      "max_players": 1,
      "difficulty": 1,
      "play_time": 1,
      "thumbnail_path": "thumbnail.png",
      "background_path": "background.jpg",
      "executable_path": "game.exe",
      "controller_support": false,
      "controls": {
        "keyboard": {
          "操作名1": "キー1",
          "操作名2": "キー2"
        },
        "gamepad": {
          "操作名1": "ボタン1",
          "操作名2": "ボタン2"
        }
      },
      "key_mapping": null
    }
```

### ステップ3：あなたのゲーム情報に書き換える

貼り付けたテンプレートの各項目を、以下の説明を参考にして、あなたのゲームの情報に書き換えてください。

| 項目名 | 説明 | 記入例 |
| :--- | :--- | :--- |
| **`game_id`** | **ゲームのID（半角英数字と `_` `-` のみ）**<br>プログラムがゲームを識別するためのIDです。**ゲームのフォルダ名と同じ**にしてください。 | `"galaxy_striker"` |
| **`title`** | **ゲームの正式名称**<br>ランチャーに表示される、あなたのゲームのタイトルです。**こちらは日本語も使えます。** | `"ギャラクシー・ストライカー"` |
| **`description`** | **ゲームの紹介文**<br>どんなゲームなのか、どこが面白いのかを来場者に伝えるための文章です。 | `"無限に迫りくる敵を撃ち落せ！..."` |
| **`developers`** | **制作者リスト**<br>あなたの姓・名と、**何期生か**を教えてください。（教員の場合は`0`）<br>チーム制作の場合は、`{ }`をカンマで区切って全員分を記入してください。 | `[{ "last_name": "山田", "first_name": "太郎", "grade": 47 }]` |
| **`release_year`** | **公開年**<br>このゲームを最初に公開した年を西暦でお願いします。 | `2025` |
| **`genre`** | **ゲームのジャンル**<br>「アクション」「パズル」など。複数ある場合はカンマで区切ってください。 | `["シューティング", "縦スクロール"]` |
| **`min_players`** | **最小プレイ人数**<br>1人専用なら`1`、2人対戦専用なら`2`になります。 | `1` |
| **`max_players`** | **最大プレイ人数**<br>1人専用なら`1`、1〜4人で遊べるなら`4`になります。 | `4` |
| **`difficulty`** | **難易度（1〜3）**<br>**1:** かんたん, **2:** ふつう, **3:** むずかしい、の3段階でお願いします。 | `2` |
| **`play_time`** | **想定プレイ時間（1〜3）**<br>ゲーム開始から終了までの**1プレイサイクル**にかかる時間の目安です。<br>**1:** 短い(〜10分), **2:** ふつう(10〜30分), **3:** 長い(30分〜) | `1` |
| **`thumbnail_path`** | **サムネイル画像のパス**<br>正方形のサムネイル画像（**SVG**形式、または512x512ピクセル以上の**PNG**）を用意してください。 | `"thumbnail.png"` |
| **`background_path`** | **背景画像・動画のパス**<br>ゲーム選択時に表示される、横長の画像または動画（1920x1080ピクセル推奨）を用意してください。 | `"background.jpg"` |
| **`executable_path`** | **実行ファイルのパス**<br>ゲームを起動するための`.exe`ファイルです。**【最重要】** | `"game.exe"` |
| **`controller_support`** | **コントローラー対応**<br>コントローラーで遊べる場合は`true`、キーボードのみの場合は`false`を記入してください。 | `true` |
| **`controls`** | **操作方法**<br>キーボードとゲームパッドの操作方法を、分かる範囲で記入してください。 | `{"keyboard": {"移動": "矢印キー"}, "gamepad": {"移動": "左スティック"}}` |

**【重要】各種ファイルのパスの書き方について**
*   **【基本】** サムネイル画像、背景、実行ファイル(`.exe`)は、この`launcher_info.json`と**同じ階層に置いて**、ファイル名だけを記入するのが最も簡単です。
*   **【応用】** もしゲームの構造上、ファイルを別のサブフォルダに置く必要がある場合のみ、`"フォルダ名/ファイル名"`の形で、**このJSONファイルからの相対パス**を記入してください。

**【補足：項目を空欄にした場合】**
もし、`description`や`background_path`などの項目に、どうしても設定する内容がない場合は、`""`のように空欄にしておいても、ランチャーがエラーで落ちることはありません。説明文は空欄で表示され、画像がない場合は「No Image」のような代替画像が表示されるように設計されています。
ですが、文化祭で最高の展示にするために、できるだけ全ての項目を埋めるように協力をお願いします！

### ステップ4：ランチャーでの表示を確認する

1.  共有フォルダにあるランチャー本体（`GCTonePrism.exe`など）を起動します。
2.  あなたのゲームがリストに正しく表示されているか、タイトルや説明文が意図通りかを確認してください。

**【トラブルシューティング】もし、あなたのゲームが表示されない、または情報がおかしい場合**
*   ランチャー実行中に **`Ctrl + Alt + F12`** を押して、**デバッグモード**を有効にしてください。
*   画面に、プログラムの動作ログが表示されます。
*   `「[あなたのゲーム名]」の情報を読み込みました` というメッセージが出ていますか？
*   もし、`警告: ...が見つかりません` や `警告: ...の解析に失敗しました` というメッセージが出ていたら、`launcher_info.json`のファイル名や、中の記述（特にカンマ`,`や引用符`"`のつけ忘れ）が間違っている可能性が高いです。テンプレートと見比べて、間違いがないか確認してみてください。

## 🔧 ランチャーの仕組み (詳しく知りたい人向け)

このセクションは、Prismがどのような考え方で設計・開発されているかを知りたい人向けの情報です。

### データ駆動設計 (Data-Driven Design)
Prismの最も重要な設計思想です。UIの表示内容やゲームの起動情報は、プログラムの中に直接書き込むのではなく、外部の**JSONファイル**によって定義されます。
*   **`launcher_config.json`:** プロジェクトのルートに配置されます。ゲームが格納されている`Games`フォルダの場所や、ゲームの表示順序 (`games_order`) など、ランチャー全体の動作を定義します。
*   **`launcher_info.json`:** 各ゲームのフォルダ内に配置されます。タイトル、説明文、サムネイルや背景画像のパス、実行ファイルのパスなど、個別のゲームに関するすべての情報を定義します。

この仕組みにより、プログラマーでなくても、テキストファイルを編集するだけでランチャーの内容を安全に変更できます。

### プロジェクトのフォルダ構造
このプロジェクトの主要なフォルダと、その役割です。

*   `assets/`: フォントや共通の画像など、プロジェクト全体で使う素材が入っています。
*   `scenes/`: Godotのシーンファイル（`.tscn`）が、役割ごとに整理されて入っています。
    *   `main/`: ユーザーが直接目にする「画面」です。（例: `menu.tscn`）
    *   `components/`: UIを構成する、再利用可能な「部品」です。（例: `game_thumbnail.tscn`）
    *   `system/`: 目には見えない、プロジェクトの根幹を支える「システム」です。（例: `global.tscn`）
    *   `tools/`: 開発を助ける「道具」です。（例: `debug_monitor.tscn`）
*   `scripts/`: 各シーンにアタッチする、GDScriptのコード（`.gd`）が入っています。

## 🚀 ランチャーの開発に参加するには？

### 前提条件
*   [Godot Engine (v4.2 or later)](https://godotengine.org/)
*   [Git](https://git-scm.com/)
*   [GitHub Desktop](https://desktop.github.com/) (推奨)

### インストールと実行

1.  **リポジトリをクローンする:** GitHub Desktopまたはコマンドラインを使って、このプロジェクト一式を自分のPCにダウンロードします。
2.  **プロジェクトを開く:** Godot Engineを起動し、「インポート」から、ダウンロードしたフォルダの中にある`project.godot`ファイルを選択します。
3.  **実行する:** `F5`キーを押すと、ランチャーが実行されます。

*参考: コマンドラインを使う場合、手順1は以下のコマンドで実行できます。*

```bash
    # 1. このリポジトリをクローンし、
    git clone https://github.com/ken1208git/GCTonePrism.git
    
    # 2. プロジェクトフォルダに移動します
    cd GCTonePrism
```

### 開発の進め方
このプロジェクトは、誰でも自由な発想で機能追加や改善ができるように、ボトムアップな開発スタイルを目指しています。

1.  **アイデアをIssueにする:** 「こんな機能があったら良いな」「ここが使いにくいな」と思いついたら、まずはGitHubの**Issues**に、あなたのアイデアを**新しいIssueとして作成**してください。
    - **[GCTonePrism - GitHub Issues](https://github.com/ken1208git/GCTonePrism/issues)**
2.  **ブランチを作成する:** `main`ブランチを直接変更せず、新しい作業を始める前には、必ず自分専用の**ブランチ（Branch）**を作成しましょう。GitHub Desktopで簡単に作成できます。
3.  **コーディングとコミット:** 作業がある程度キリの良いところまで進んだら、変更内容を**コミット（Commit）**して、記録に残します。
4.  **プッシュとプルリクエスト:** 作業が完了したら、自分のブランチをGitHubに**プッシュ（Push）**し、`main`ブランチへの**プルリクエスト（Pull Request）**を作成して、他のメンバーにレビューを依頼します。

**【重要】**
この説明書きは、開発の基本的な流れを示すものです。GitやGitHubの詳しい使い方（ブランチの切り方、コンフリクトの解消方法など）については、ここではすべてを説明しきれません。**分からないことがあれば、先輩に聞いたり、Webで検索したりして、各自で積極的に調べる姿勢を大切にしてください。**

### デバッグモード
ランチャー実行中に **`Ctrl + Alt + F12`** を押すと、画面にデバッグ情報（FPS、メモリ使用量、ログなど）が表示されるオーバーレイを切り替えることができます。開発中に問題が発生した場合は、この情報を活用してください。
